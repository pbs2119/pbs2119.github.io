Now that I’m (sort of) up to speed on the codebase, I’ve been asked to update the graphical user interface pipeline for the system.  As a brief bit of background, the MBABench project uses three different pipelines to send example spreadsheet tasks to LLMs for completion.  The GUI version automates a web browser using an existing framework called Playwright to send the task through the normal consumer-facing UI most people have probably used to interact with an LLM.  

Although Playwright has been around for a couple of years, it’s new to me!  It’s been pretty neat to see how some python code and the command line interface can drive a browser window.  These scripts can break if the UI changes (which seems to happen frequently with the LLM providers) and that is what I’m focused on.  

One thing I’ve noticed pretty quickly - I’m burning through tokens at a far faster rate than ideal during my test runs of the pipeline.  This hasn’t been much of an issue before for my personal use of LLMs, but I can see why costs are quickly becoming a big concern for those doing research or engineering work with an LLM at scale.
