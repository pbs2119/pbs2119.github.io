Because I’m joining a project that is already underway, the first order of business has been getting up to speed on the existing codebase.  For those interested, the GitHub repo for the phase 1 version the team completed before I joined can be found here:

https://github.com/namkoong-lab/MBABench

It’s been both challenging and rewarding getting to see how a real system with a lot of pieces and moving parts is constructed.  It’s very different from anything I’ve learned in a classroom, where you tend to focus on toy problems that typically all could fit within a single Jupyter notebook. 

Something I’ve been asked to create is a pipeline that can feed example spreadsheet tasks (consisting of instructions and any preliminary data) into an LLM to forecast the amount of time it would take a human to complete the task.  It’ll be a good opportunity to get some exposure to using LLM APIs instead of the graphical user interface I’m accustomed to.
